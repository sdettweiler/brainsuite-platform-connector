## Tech Debt

### Specification Mismatches (non-blocking)

**PERF-01 minimum guard:** `_compute_performer_tag` uses `total_scored < 3` (dashboard.py:43). Requirement specifies 10. Badges appear earlier than specified. Low risk — affects only small orgs with 3–9 scored assets.

**TREND-02 proxy implementation:** Score trend uses `creative_score_results.scored_at` as a proxy for score history. Each asset has one data point (its last scoring date). No append-only history. Acceptable for v1.1 since BrainSuite scores are static (D-09), but misleadingly labelled "Score Trend." If re-scoring is ever enabled, this will need a proper creative_score_history table.

### Missing Artifacts

- **VERIFICATION.md** for Phases 7, 8, 10 — no formal verifier pass
- **VALIDATION.md** for Phase 10 — Nyquist validation not done
- **Nyquist compliance** — all 5 existing VALIDATION.md files in `draft` status

### Minor Code Issues

- `get_asset_detail()` hardcoded None for scoring fields (dashboard.py:597-599) — confusing but not used by frontend
- `asyncio.create_task()` for notifications has latent GC risk (pre-existing pattern; acceptable)
- COMPLETE guard in `_autofill()` has narrow race window on concurrent calls (only reachable if backfill triggers auto-fill — not current behavior)
