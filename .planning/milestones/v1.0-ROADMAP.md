# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-03-25
**Phases:** 1–4
**Total Plans:** 19

## Overview

Starting from a brownfield Angular 17 + FastAPI application already syncing creatives from four ad platforms, this milestone closed four gaps in sequence: first decoupled the platform from Replit into a portable Docker Compose stack; then fixed production security vulnerabilities blocking external user onboarding; then wired in the BrainSuite scoring pipeline (the primary milestone deliverable); finally polished the dashboard UX and surfaced sync reliability so agencies can trust the data they see.

- [x] 03-05: Frontend score badge, polling, Creative Effectiveness tab

**Success Criteria (all met):**
1. After a platform sync completes, newly imported creatives appear with a scoring status indicator and receive a BrainSuite score within 15 minutes without any manual action
2. Each creative in the dashboard displays a score badge; clicking the badge opens a dimension breakdown panel with all scoring dimensions returned by the BrainSuite API
3. A user can trigger a manual re-score on any individual creative via the UI and see the score update after the next scheduler run
4. BrainSuite API failures (429, 5xx) do not crash the scheduler or leave assets permanently stuck — failed assets retry automatically; 4xx failures mark the asset as permanently failed with a visible reason
5. The frontend polling endpoint (`/scoring/status`) is only called while PENDING or PROCESSING assets are on screen — not on every page load

## Milestone Summary

**Delivered:** A production-ready multi-tenant platform connector that syncs ad creatives from Meta, TikTok, Google Ads, and DV360, automatically scores them via BrainSuite, and surfaces performance metrics + effectiveness scores in a unified dashboard with sort, filter, and health monitoring.

**Key Accomplishments:**
1. Full Docker Compose portability — application runs anywhere with `docker compose up`, zero Replit dependency
2. Production security hardened — httpOnly cookie auth, Redis OAuth sessions, encrypted tokens, path traversal fix, typed DTOs
3. BrainSuite scoring pipeline — async UNSCORED→PENDING→PROCESSING→COMPLETE state machine with tenacity retry, 15-min batch scheduler, score + dimension breakdown UI
4. Dashboard polish — score range slider, video thumbnail fallback, nullslast sort, score badge + Creative Effectiveness tab
5. Platform reliability — health badges, reconnect prompts, token_expiry exposure, SCHEDULER_ENABLED guard for multi-worker deployments

**Decimal Phases:** None

**Key Decisions:**
- Replaced Replit sidecars with MinIO (pinned RELEASE.2025-10-15T17-29-55Z) + Redis Docker Compose services
- httpOnly cookie for refresh token; in-memory only for access token (XSS prevention)
- Session-per-operation pattern for BrainSuite scoring (never hold DB session during HTTP calls)
- on_conflict_do_nothing for UNSCORED injection (prevents re-sync resetting completed scores)
- ngx-slider pinned to 17.0.2 (Angular 17 compatible)
- SCHEDULER_ENABLED env var guard (multi-worker Autoscale / cloud deployments)

**Technical Debt:**
- BrainSuite scoring credentials not yet configured in production — code path fully implemented, pending credential setup
- Google Ads OAuth consent screen "Published" status should be verified before external user onboarding (Testing mode = 7-day refresh token expiry)
