Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, score badge inner spacing, `margin-left: 4px` on `.scoring-label` |
| sm | 8px | Compact element spacing, chip padding, gap between filter elements |
| md | 16px | Card padding (`.card`), filter bar inner padding, default element spacing |
| lg | 24px | Section padding (`.section-header` padding), page-level gaps |

Exceptions:
- Touch targets for icon-only buttons: 36px (existing `.platform-filter-btn` width/height) — not 44px; this project uses 36px for density consistency with Angular Material density: 0.

- Dashboard card thumbnail height: 160px (locked by existing `.tile-thumb` style and D-08).

Source: `frontend/src/styles.scss` utility classes (`.gap-2` = 8px, `.gap-3` = 12px, `.gap-4` = 16px, `.gap-6` = 24px) and component SCSS.

## Slider Styling Contract

The ngx-slider must match the existing orange accent theme. Apply these overrides in `dashboard.component.ts` inline styles or a `:host ::ng-deep` block:

| Property | Value | Rationale |
|----------|-------|-----------|
| Track color (active/filled) | `var(--accent)` = `#FF7700` | Matches accent reserved list — D-04 / CONTEXT.md |
| Track color (inactive) | `var(--border)` = `#404040` (dark) / `#C2CDE2` (light) | Consistent with input field borders |
| Handle background | `#FFFFFF` | High contrast against dark track |
| Handle border | `2px solid var(--accent)` | Reinforces accent theme |
| Handle diameter | 20px | Thumb target slightly above 16px default for usability |
| Bubble (value tooltip) background | `var(--bg-card)` | Matches existing tooltip/popover style |
| Bubble text color | `var(--text-primary)` | Readable on card background |
| Disabled state opacity | 0.4 | Matches `disabled` pattern used elsewhere in the app (D-05) |
| Disabled tooltip | "No scored creatives yet" | D-05 — shown via `matTooltip` on slider wrapper |
| Debounce | 400ms | `debounceTime(400)` on `Subject<void>` (RESEARCH.md Pattern 1) |

## Health Badge Contract

Each platform connection row in the Configuration page gains three new data points:

| Health State | Badge Class | Badge Label | Action Button |
|-------------|-------------|-------------|---------------|
| `connected` | `badge badge-success` | "Connected" | None |
| `token_expired` | `badge badge-warning` | "Token expired" | "Reconnect Account" (accent fill button) |
| `sync_failed` | `badge badge-error` | "Sync failed" | "Reconnect Account" (accent fill button) |

Health state computation (client-side, D-15):
- `token_expired`: `token_expiry` field is set AND is before `new Date()`
- `sync_failed`: `last_synced_at` is null OR hours since last sync > 48
- `connected`: neither of the above

Threshold rationale (Claude's Discretion from CONTEXT.md): 48 hours is the sync-failed threshold. Ad platform data typically refreshes daily; 48 hours provides a one-missed-sync buffer before surfacing the error.

Last sync display: `formatDistanceToNow(new Date(conn.last_synced_at), { addSuffix: true })` from `date-fns` (already in `package.json`). Shows "Never" when `last_synced_at` is null.

Column placement: Replace (not augment) the existing `.col-status` column with the new health badge. The existing `sync_status` chip (ACTIVE/EXPIRED/ERROR/PENDING) is superseded by the richer health badge. Column width: 130px (same as `.col-status` currently). Add `.col-last-sync` at 130px for relative time text.

## Thumbnail Fallback Contract

| Creative type | Condition | Rendered output |
|--------------|-----------|-----------------|
| Image | `asset_url` or `thumbnail_url` present | `<img>` with `object-fit: cover` in 160px container |
| Image | Both null | `placeholder.svg` (D-07) |
| Video | `thumbnail_url` present | `<img>` with `object-fit: cover` in 160px container |
| Video | `thumbnail_url` null | Dark bg (`#111`) + platform icon (48x48px, opacity 0.6) centered + "VIDEO" tag (9px/600, bottom-right 6px) |

The `getTileThumbnail()` method returns `null` for the video-no-thumb case; template switches to `.video-fallback` div. CSS class `.video-no-thumb` sets `background: #111` on the containing tile.

Image tile height: 160px (locked, D-08). No change to existing `object-fit: cover` constraint.

## Interaction States

### Score Range Slider

| State | Visual |
|-------|--------|
| Default (0–100, full range) | Both handles at extremes; track fully `var(--accent)` colored |
| Filtered (handles moved inward) | Active range highlighted in `var(--accent)`; both value bubbles visible |
| Disabled (no scored assets) | Opacity 0.4; `matTooltip="No scored creatives yet"` on wrapper div |
| Loading (API in-flight after debounce) | Dashboard grid shows `.skeleton` tiles (existing shimmer pattern) |

### Health Badge + Reconnect

| State | Visual |
|-------|--------|
| Connected | Green `.badge-success` pill, no button |
| Token expired | Amber `.badge-warning` pill + "Reconnect Account" `mat-button` with `var(--accent)` color |
| Sync failed | Red `.badge-error` pill + "Reconnect Account" `mat-button` with `var(--accent)` color |
| Reconnect in-flight | Button shows `mat-spinner` (diameter 16px) inline, disabled |

### Sort (Score column)

| State | Visual |
|-------|--------|
| Score sort active (asc) | Sort direction icon rotated; scored items sorted asc, unscored assets always at bottom (NULLS LAST) |
| Score sort active (desc) | Highest-scored items first; unscored assets always at bottom |
| Score sort inactive | No change from current sort indicator |
