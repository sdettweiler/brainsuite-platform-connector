<decisions_made>

- `visualizations` at kpi level is an ARRAY `[{type, url}]` per API docs ŌĆö not a dict. Both persist function and frontend were updated to handle this.
- S3 path for viz files: `creatives/brainsuite/{asset_id}/viz/{breadcrumb}/{idx}` ŌĆö must start with `creatives/` because `serve_object` route in `main.py` prepends `creatives/` for paths not already starting with it.
- Re-fetch uses `BackgroundTasks` (FastAPI), not APScheduler ŌĆö fires immediately on request, no queue needed since it's a quick GET to BrainSuite.
- `getActiveViz()` helper introduced to avoid repeated `Array.find()` in template.
- `assetIsVideo()` helper checks for `VIDEO` in `asset_format` string (case-insensitive, includes substring) to be robust.
</decisions_made>
