**1. Create `backend/app/services/sync/scoring_job.py`:**

```python
"""BrainSuite scoring batch job — runs via APScheduler every 15 minutes."""
import logging
from datetime import datetime, timezone
from sqlalchemy import select
from sqlalchemy.orm import selectinload

from app.db.base import get_session_factory
from app.models.creative import CreativeAsset, AssetMetadataValue
from app.models.scoring import CreativeScoreResult
from app.models.metadata import MetadataField
from app.services.brainsuite_score import (
    brainsuite_score_service,
    build_scoring_payload,
    extract_score_data,
    BrainSuiteJobError,
)
from app.services.object_storage import get_object_storage

logger = logging.getLogger(__name__)

BATCH_SIZE = 20
```

**`async def run_scoring_batch() -> None`:**

Phase 1 — Query batch (inside a DB session, then release it):
- Open `async with get_session_factory()() as db:`
- SELECT up to `BATCH_SIZE` `CreativeScoreResult` rows WHERE `scoring_status == "UNSCORED"` joined with `CreativeAsset` WHERE `asset_format == "VIDEO"`, ordered by `created_at ASC`, limit `BATCH_SIZE`
- For each row, eagerly load the associated `CreativeAsset` (use `selectinload` or join)
- Collect a list of dicts: `{"score_id": row.id, "asset_id": row.creative_asset_id, "asset": <CreativeAsset>}`
- Set all selected rows to `scoring_status = "PENDING"`, commit, then **close the session** (exit the `async with` block)
- If batch is empty, return early with log message

Phase 2 — Process each asset (NO DB session held during HTTP calls):
- For each item in the batch:
  - `try:`
    - Get the asset's S3 key from `asset.asset_url`. Strip `/objects/` prefix if present: `s3_key = asset.asset_url.lstrip("/objects/") if asset.asset_url and asset.asset_url.startswith("/objects/") else asset.asset_url`
    - If `s3_key` is None or empty: skip, mark FAILED with `error_reason = "No S3 asset URL available"`
    - Generate fresh signed URL: `signed_url = get_object_storage().generate_signed_url(s3_key, ttl_sec=3600)`
    - If signed_url is None: skip, mark FAILED with `error_reason = "Failed to generate signed S3 URL"`
    - Load metadata for this asset: open a **new** DB session, query `AssetMetadataValue` joined with `MetadataField` WHERE `asset_id == asset.id` and `field.name LIKE 'brainsuite_%'`, build a `dict[str, str]` mapping `field.name -> value.value`, close session
    - Build payload: `build_scoring_payload(asset.ad_name or f"{asset.id}.mp4", signed_url, asset.platform, asset.placement, metadata_dict)`
    - Submit job: `job_response = await brainsuite_score_service.create_job_with_retry(payload)`
    - Extract `job_id` from response (key: `"id"` or `"jobId"` — check the response shape)
    - Update score record: open new session, set `brainsuite_job_id = job_id`, `scoring_status = "PROCESSING"`, commit, close
    - Poll: `result = await brainsuite_score_service.poll_job_status(job_id)` — **NO DB session open**
    - Extract scores: `score_data = extract_score_data(result)`
    - Write results: open new session, update `CreativeScoreResult` — set `total_score = score_data["total_score"]`, `total_rating = score_data["total_rating"]`, `score_dimensions = score_data["score_dimensions"]`, `scoring_status = "COMPLETE"`, `scored_at = datetime.now(timezone.utc)`, `updated_at = datetime.now(timezone.utc)`, commit, close
  - `except BrainSuiteJobError as e:` — mark FAILED with `error_reason = str(e)[:500]`
  - `except Exception as e:` — mark FAILED with `error_reason = f"{type(e).__name__}: {str(e)[:500]}"`, log error with asset ID context
  - For any failure case: open new session, update the `CreativeScoreResult` row (by `score_id`) setting `scoring_status = "FAILED"`, `error_reason`, `updated_at`, commit, close

Key constraints:
- NEVER hold a DB session during `create_job_with_retry` or `poll_job_status` (Pitfall 4 from RESEARCH.md)
- Use `max_instances=1` on the APScheduler job (Pitfall 5)
- Treat `Stale` BrainSuite status as `Failed` (Pitfall 3 — handled inside poll_job_status)
