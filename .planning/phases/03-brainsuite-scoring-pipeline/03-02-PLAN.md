class BrainSuiteRateLimitError(Exception):
    def __init__(self, reset_at: datetime):

**2. BrainSuiteScoreService class with:**

`__init__(self)`: Initialize `_token: str | None = None`, `_token_expires_at: datetime | None = None`.

`async def _get_token(self) -> str`:
- Check if cached token is still valid (now < _token_expires_at)
- If expired or missing: POST to `settings.BRAINSUITE_AUTH_URL` with Basic auth header (base64 encode `{BRAINSUITE_CLIENT_ID}:{BRAINSUITE_CLIENT_SECRET}`), body `grant_type=client_credentials`, Content-Type `application/x-www-form-urlencoded`
- Parse response `{"access_token": "...", "token_type": "Bearer"}`
- Cache token with 50-minute expiry (`_token_expires_at = now + timedelta(minutes=50)`)
- On 401 from any subsequent API call: invalidate cache (`_token = None`) and re-fetch

`async def _create_job_raw(self, token: str, payload: dict) -> dict`:
- POST to `{settings.BRAINSUITE_BASE_URL}/v1/jobs/ACE_VIDEO/ACE_VIDEO_SMV_API/create`
- Headers: `Authorization: Bearer {token}`, `Content-Type: application/json`
- Timeout: 60 seconds
- On 429: parse `x-ratelimit-reset` header (ISO 8601 UTC), raise `BrainSuiteRateLimitError(reset_at)`
- On 5xx: raise `BrainSuite5xxError`
- On 4xx (not 429): raise `ValueError` with status code and response text
- On 2xx: return `resp.json()`

`async def create_job_with_retry(self, payload: dict) -> dict`:
- Loop up to 5 attempts
- On `BrainSuiteRateLimitError`: compute `wait_secs = max(0, (reset_at - now_utc).total_seconds()) + 2`, log warning, `await asyncio.sleep(wait_secs)`
- On `BrainSuite5xxError`: exponential backoff `min(2 ** attempt * 5, 120)` seconds
- On 401 response during `_create_job_raw`: invalidate token (`self._token = None`), retry with fresh token
- Exhaust retries: raise `RuntimeError("BrainSuite create_job exhausted retries")`

`async def poll_job_status(self, job_id: str, max_polls: int = 60, poll_interval: int = 30) -> dict`:
- GET `{settings.BRAINSUITE_BASE_URL}/v1/jobs/{job_id}` with Bearer token
- Loop up to `max_polls` iterations, sleeping `poll_interval` seconds between each
- Terminal statuses: `Succeeded`, `Failed`, `Stale`
- While status is `Scheduled`, `Created`, or `Started`: continue polling
- On `Succeeded`: return full response JSON
- On `Failed` or `Stale`: raise `BrainSuiteJobError` with status and any error detail
- On max_polls exhausted: raise `BrainSuiteJobError("Job polling timed out")`
