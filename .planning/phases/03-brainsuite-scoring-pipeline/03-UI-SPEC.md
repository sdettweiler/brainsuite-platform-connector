## Interaction Contract

### Polling Behavior (SCORE-08)

- Frontend polls `GET /api/v1/scoring/status?asset_ids=...` every **10 seconds** using `interval(10000)` + `takeUntil` when any visible asset is `PENDING` or `PROCESSING`.
- Polling stops automatically when zero `PENDING`/`PROCESSING` assets remain in the current page view.
- No visual polling indicator beyond the individual spinner per asset row.

### Re-score Flow

1. User right-clicks asset row → context menu appears
2. User clicks "Score now"
3. `POST /api/v1/scoring/{asset_id}/rescore` fires
4. On success: score badge immediately switches to `PENDING` spinner; polling starts
5. Toast appears: "Scoring queued — results in ~2 minutes"
6. On error: toast appears: "Could not queue scoring. Try again."

### Metadata Validation (pre-requisite state)

If `brainsuite_brand_names` or `brainsuite_asset_language` metadata fields are missing for an asset when "Score now" is triggered:

- Show `MatSnackBar` warning: "Required fields missing — add Brand Names and Language in asset details first."
- Do NOT call the rescore endpoint.
- Duration: 5000ms, `panelClass: 'snackbar-warning'`

## Accessibility Contract

- Score badge: `aria-label="Score: {value}, {rating}"` on the badge element
- Spinner: `aria-label="Scoring in progress"` on `<mat-spinner>`
- Dash for unscored: `aria-label="Not yet scored"`
- Context menu item "Score now": standard `MatMenuItem` keyboard navigation (Enter/Space to trigger)
- All toast messages: announced via Angular Material `MatSnackBar` (live region built-in)
- Rating color must not be the sole differentiator — numeric score value always shown alongside color
