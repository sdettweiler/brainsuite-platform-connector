    **Constants:**

    ```python
    AUTO_FILL_TYPE_VISION = {"language", "brand_names"}

    ```

    **Main function `run_autofill_for_asset(asset_id: uuid.UUID, org_id: uuid.UUID) -> None`:**
    - Wraps `_autofill()` in try/except; on exception logs with `logger.exception()` and calls `_set_status(asset_id, "FAILED")`

    **`_autofill(asset_id, org_id)` — 4-phase pattern:**
    - Phase 1 (DB read): Insert tracking row with `pg_insert(...).on_conflict_do_nothing(index_elements=["asset_id"])`. Then SELECT the tracking row — if `ai_inference_status == "COMPLETE"`, return immediately (COMPLETE guard per AI-06). Query `MetadataField` where `organization_id == org_id AND auto_fill_enabled == True AND is_active == True`. If no fields, set status COMPLETE and return. Load `CreativeAsset` and extract `asset_format`, `asset_url`, `campaign_name`, `ad_name`. Close session.
    - Phase 2 (asset download): If any field needs vision or audio, download asset binary from MinIO using `get_object_storage().download_blob(s3_key)`. The `s3_key` is derived from `asset_url` by stripping leading `/` and removing `objects/` prefix (same pattern as existing scoring code).
    - Phase 3 (inference): If `settings.OPENAI_API_KEY` is None, skip all OpenAI calls (per D-09). Otherwise:
      - For vision fields: If asset is IMAGE, use raw bytes. If VIDEO, extract first frame using `_extract_first_frame(asset_bytes)` via imageio-ffmpeg (Pattern 6 from RESEARCH.md). Downsample if >4MB using `_downsample_image(image_bytes, content_type)` with `Pillow Image.thumbnail((1568, 1568), Image.LANCZOS)`. Call `_run_vision(image_bytes, content_type)` using `AsyncOpenAI.beta.chat.completions.parse()` with model `"gpt-4o-mini"` and Pydantic `VisionResult(language: str | None, brand_names: list[str])`.
      - For audio fields: Extract audio from video via `_extract_audio_bytes(video_bytes)` using ffmpeg subprocess with `imageio_ffmpeg.get_ffmpeg_exe()` as the binary (not system ffmpeg). Call `_run_whisper(audio_bytes)` using `AsyncOpenAI.audio.transcriptions.create(model="whisper-1", response_format="verbose_json")`. Return `{"text": transcript.text, "language": transcript.language}`.
    - Phase 4 (DB write): Route values by `auto_fill_type`:
      - `"language"` -> `vision_result.get("language") or default_value`
      - `"brand_names"` -> `", ".join(vision_result.get("brand_names", [])) or default_value`
      - `"vo_transcript"` -> `audio_result.get("text", "")[:2000] or default_value`
      - `"vo_language"` -> `audio_result.get("language") or default_value`
      - `"campaign_name"` -> `campaign_name or default_value`
      - `"ad_name"` -> `ad_name or default_value`
      - `"fixed_value"` -> `default_value`
      Upsert each value into AssetMetadataValue (SELECT existing, update if exists, INSERT if not). Set tracking status to COMPLETE.

    **`_set_status(asset_id, status)` helper:** Opens new session, updates tracking row status + updated_at, commits.

    **`_downsample_image(image_bytes, content_type) -> bytes`:** If `len(image_bytes) > 4 * 1024 * 1024`, open with PIL, thumbnail to (1568, 1568) LANCZOS, save to BytesIO as JPEG (or PNG if content_type is image/png), return bytes. Otherwise return original bytes.

    **`_extract_first_frame(video_bytes) -> bytes | None`:** Use `imageio_ffmpeg.read_frames()` with BytesIO input, extract first frame, convert raw RGB to JPEG via PIL. Return JPEG bytes or None.

    **`_extract_audio_bytes(video_bytes) -> bytes | None`:** Use `asyncio.create_subprocess_exec()` with `imageio_ffmpeg.get_ffmpeg_exe()` as the binary. Args: `-i pipe:0 -vn -acodec pcm_s16le -ar 16000 -ac 1 -f wav pipe:1`. Pipe video_bytes to stdin, read stdout. Return None if returncode != 0 or stdout < 1000 bytes.
