## Interaction Contracts

### 1. Metadata Config Page — Auto-Fill Toggle + Type Selector

**Location:** `configuration/pages/metadata.component.ts` — inside each expanded field-row

**States:**
- `auto_fill_enabled = false` (default): toggle is OFF; `auto_fill_type` selector is hidden
- `auto_fill_enabled = true`, `auto_fill_type = null`: toggle is ON; selector is visible with placeholder "Select inference type"; field-row has a subtle `--accent-light` left-border (4px, accent color) to indicate AI-enabled state
- `auto_fill_enabled = true`, `auto_fill_type` set: toggle ON; selector shows selected value; save triggers `PATCH /assets/metadata/fields/{id}`

**Toggle interaction:**
- Toggle ON: `auto_fill_type` selector fades in with `opacity 0.15s ease`, no layout shift
- Toggle OFF: selector fades out; `auto_fill_type` is cleared to `null` before save
- Saving state: inline `mat-spinner` diameter 16 replaces save button label text "Saving…"
- Success: `MatSnackBar` "Auto-fill settings saved" duration 2000ms
- Error: `MatSnackBar` "Failed to save — please try again" duration 4000ms

**Auto-fill type options (exact copy for `mat-select` `mat-option` labels):**

| Value | Display Label |
|-------|--------------|
| `language` | Language (detected from content) |
| `brand_names` | Brand Names (extracted from creative) |
| `vo_transcript` | Voice Over Transcript (Whisper) |
| `vo_language` | Voice Over Language (Whisper) |
| `campaign_name` | Project Name (from campaign data) |
| `ad_name` | Asset Name (from ad data) |
| `fixed_value` | Fixed Value (always use default) |

**Note on `campaign_name` and `ad_name`:** These are deterministic (sync-data fill, no OpenAI). `fixed_value` triggers no inference — uses `default_value` only. The selector displays all seven options regardless; the distinction is backend-handled.

### 2. Asset Detail Dialog — Inference Status Indicator

**Location:** Metadata tab or metadata section of `asset-detail-dialog.component.ts`

**Display rule:** If the asset has an `ai_inference_status` value (from the execution tracking row), render an `InferenceStatusBadge` in the metadata section header row.

**Badge copy and variant:**

| Status | Badge Label | Badge Variant | Icon |
|--------|-------------|---------------|------|
| `PENDING` | "AI analysis running…" | `badge-warning` | `bi-hourglass-split` (12px, animated spin optional) |
| `COMPLETE` | "AI auto-filled" | `badge-success` | `bi-check-circle` (12px) |
| `FAILED` | "AI analysis failed — will retry on next sync" | `badge-error` | `bi-exclamation-circle` (12px) |
| `null` / absent | (badge not rendered) | — | — |

**No polling in dialog.** Status is loaded once when the dialog opens (current GET /assets/{id} response). The `PENDING` badge communicates that auto-fill ran or is running — it does not animate indefinitely or poll. If the user reopens the dialog after the next sync, status will reflect the updated result.

### 3. Manual Metadata Edit → Rescore Reset Feedback

**Location:** Metadata field inputs in `asset-detail-dialog.component.ts`

**Current behavior (unchanged):** User edits a field value → PATCH saves → scoring_status resets to `UNSCORED` server-side.

**New Phase 9 addition:** After a successful metadata PATCH that resets scoring status, show a `MatSnackBar` toast: "Metadata saved — creative queued for rescoring." Duration: 3000ms. This is informational only; no spinner or blocking state.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (auto-fill toggle label) | "Enable Auto-fill" |
| Auto-fill type selector placeholder | "Select inference type" |
| Config page toggle section label | "AI Auto-Fill" |
| Config page toggle helper text | "Fields are filled automatically during asset sync using AI inference." |
| PENDING status badge | "AI analysis running…" |
| COMPLETE status badge | "AI auto-filled" |
| FAILED status badge | "AI analysis failed — will retry on next sync" |
| Empty auto-fill type (toggle ON, no type selected) warning | "Select an inference type to enable auto-fill for this field." |
| Save success toast | "Auto-fill settings saved" |
| Save error toast | "Failed to save — please try again" |
| Metadata edit + rescore toast | "Metadata saved — creative queued for rescoring" |
| Delete field confirmation (existing pattern, unchanged) | `confirm()` dialog: "Delete field \"{label}\"? This will remove all values from assets." |

**Language rules:**
- Never mention "Claude", "Anthropic", "GPT", "OpenAI", or "Whisper" in any user-facing string. AI inference provider is invisible to the end user.
- "Auto-fill" is always hyphenated in UI labels (not "autofill" or "auto fill").
- Status badge copy uses sentence case, not title case.
- All error messages include a next-step hint ("will retry", "please try again").

| Inference status badge | No animation — renders static on dialog open |
| Page enter (if applicable) | `.page-enter` keyframe (`opacity 0→1, translateY 8px→0`) — already global |
