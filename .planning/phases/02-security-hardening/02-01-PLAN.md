**SEC-03 — Fernet key startup validation (config.py + security.py):**

In `config.py` `Settings` class:
1. Change `TOKEN_ENCRYPTION_KEY: Optional[str] = None` to `TOKEN_ENCRYPTION_KEY: str` (required, no default).
2. Add a Pydantic v2 field validator:
```python
from pydantic import field_validator

@field_validator("TOKEN_ENCRYPTION_KEY")
@classmethod
def validate_fernet_key(cls, v: str) -> str:
    if not v:
        raise ValueError("TOKEN_ENCRYPTION_KEY is required")
    try:
        from cryptography.fernet import Fernet
        Fernet(v.encode() if isinstance(v, str) else v)
    except Exception as exc:
        raise ValueError(
            f"TOKEN_ENCRYPTION_KEY is invalid (must be 32 url-safe base64 bytes): {exc}"
        ) from exc
    return v
```

In `security.py`:
1. Remove the `get_fernet()` function's fallback logic that silently generates a throwaway key.
2. Replace with direct `fernet = Fernet(settings.TOKEN_ENCRYPTION_KEY.encode())` at module level, since validation already passed at config load time.
3. Remove the `except Exception` catch (the one identified as the Fernet silent fallback bug in QUAL-04).

**SEC-04 — Path traversal fix (main.py):**

In `main.py` `serve_object()` function (around line 132):
1. Add path validation before using `object_path`:
```python
from pathlib import PurePosixPath

@app.get("/objects/{object_path:path}", include_in_schema=False)
async def serve_object(object_path: str):
    # Validate: no path traversal
    normalized = PurePosixPath(object_path)
    if ".." in normalized.parts:
        raise HTTPException(status_code=400, detail="Invalid asset path")
    # URL-decode and re-check
    from urllib.parse import unquote
    decoded = unquote(object_path)
    normalized_decoded = PurePosixPath(decoded)
    if ".." in normalized_decoded.parts:
        raise HTTPException(status_code=400, detail="Invalid asset path")
    # ... rest of function unchanged, using object_path ...
```
This catches both raw `../` and URL-encoded `%2e%2e%2f` variants. Using `PurePosixPath` instead of `Path.resolve()` avoids filesystem access and works in any environment.

def get_redirect_uri_from_request(request, platform: str) -> str:
