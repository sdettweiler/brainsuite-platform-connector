<objective>
Migrate the refresh token from JSON response body to httpOnly cookie on the backend side. This is the backend half of SEC-02 — the frontend changes follow in Plan 04.

Purpose: Prevents XSS attacks from stealing refresh tokens via JavaScript. The refresh token becomes inaccessible to JS, stored only in an httpOnly cookie scoped to auth endpoints.
Output: Modified auth.py (login/refresh/logout endpoints), updated TokenResponse schema, passing tests.
</objective>

**auth.py — /auth/login endpoint:**

1. Add imports:
```python
from fastapi import Response, Cookie
```

2. Modify the login endpoint to set an httpOnly cookie instead of returning refresh_token in the body:
```python
@router.post("/login", response_model=TokenResponse)
async def login(payload: LoginRequest, response: Response, db: AsyncSession = Depends(get_db)):
    # ... existing credential validation and token creation logic ...
    # (keep all existing logic for finding user, verifying password, creating tokens)

    # Set refresh token as httpOnly cookie
    response.set_cookie(
        key="refresh_token",
        value=refresh_token,  # the generated refresh token string
        httponly=True,
        secure=not settings.DEBUG,  # False in local dev (HTTP), True in prod (HTTPS)
        samesite="lax",
        max_age=settings.REFRESH_TOKEN_EXPIRE_DAYS * 86400,  # convert days to seconds
        path="/api/v1/auth",  # scope cookie to auth endpoints only
    )

    # Return only access_token in JSON body (no refresh_token)
    return {"access_token": access_token, "token_type": "bearer"}
```

If `settings.REFRESH_TOKEN_EXPIRE_DAYS` doesn't exist, check the existing code for the refresh token expiry value (likely `7` days or similar) and use that value directly (e.g., `max_age=7 * 86400`).

**auth.py — /auth/refresh endpoint:**

Replace the request body parameter with a Cookie parameter:
```python
@router.post("/refresh", response_model=TokenResponse)
async def refresh_token_endpoint(
    response: Response,
    refresh_token: str | None = Cookie(default=None),
    db: AsyncSession = Depends(get_db),
):
    if not refresh_token:
        raise HTTPException(status_code=401, detail="No refresh token")

    # ... existing refresh token validation logic ...
    # (keep logic for decoding token, checking DB, creating new tokens)

    # Set new refresh token cookie (token rotation)
    response.set_cookie(
        key="refresh_token",
        value=new_refresh_token,
        httponly=True,
        secure=not settings.DEBUG,
        samesite="lax",
        max_age=settings.REFRESH_TOKEN_EXPIRE_DAYS * 86400,
        path="/api/v1/auth",
    )

    return {"access_token": new_access_token, "token_type": "bearer"}
```

Remove `payload: RefreshRequest` from the function signature. Remove the line that reads `refresh_token` from `payload.refresh_token`.
