Add method `loadCorrelationData()`:
- Calls `this.api.get<CorrelationAsset[]>('/dashboard/correlation-data', { date_from: this.dateFrom, date_to: this.dateTo, platforms: this.platformFilter || undefined })`

Add method `buildScatterChart()`:
- Filters `this.correlationAssets` to: `roas !== null` (exclude null ROAS per D-11), `(spend ?? 0) >= this.correlationMinSpend` (spend threshold per D-09/D-10)
- If 0 eligible assets: sets `this.scatterOptions = {}` and returns (empty state handled in template)
- Computes 99th percentile ROAS cap: sort ROAS values ascending, `roasCap = roasValues[Math.floor(roasValues.length * 0.99)] ?? roasValues[roasValues.length - 1] ?? 1`
- Computes median score: sort scores ascending, standard median formula with even/odd guard
- Computes median ROAS: same median formula on ROAS values
- Guards: if 1 eligible asset, use its values as medians
- Builds scatter data array: `[score, Math.min(roas, roasCap), assetRef][]` — capped assets plotted at cap value
- Dot color callback per quadrant:
  - Stars (score >= medianScore AND roas >= medianRoas): `#FF7700` (var(--accent))
  - Workhorses (score < medianScore AND roas >= medianRoas): `#F39C12`
  - Question Marks (score >= medianScore AND roas < medianRoas): `#4285F4`
  - Laggards (score < medianScore AND roas < medianRoas): `#707070`
- Zero-ROAS dots: use `#707070` at 60% opacity (handled by the color callback — roas=0 < medianRoas puts them in Laggards or Question Marks quadrant naturally; no special case needed beyond the standard quadrant assignment)
- MarkLine data: `[{ xAxis: medianScore }, { yAxis: medianRoas }]` with `lineStyle: { color: '#404040', type: 'dashed', width: 1 }`, plus `{ yAxis: roasCap, lineStyle: { color: 'rgba(255,119,0,0.4)', type: 'dashed' } }`
- Sets `this.scatterOptions` to the full EChartsOption object per the research doc pattern
