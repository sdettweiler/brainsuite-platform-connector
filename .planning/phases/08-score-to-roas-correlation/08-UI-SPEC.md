## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg-primary)` — #1B1B1B dark / #F5F6FA light | Page background, drawer overlay backdrop |
| Secondary (30%) | `var(--bg-card)` — #2C2C2C dark / #FFFFFF light | Drawer panel background, tooltip background, stat tiles |
| Accent (10%) | `var(--accent)` — #FF7700 | Avg ROAS tile hover state, scatter dot fill for Stars quadrant, spend threshold input focus border |
| Destructive | `var(--error)` — #E74C3C | Not used in this phase |

Accent reserved for:
1. Avg ROAS stat tile hover border (`border-color: var(--accent)`) and cursor affordance
2. Stars quadrant scatter dots (top-right: high score + high ROAS) — full accent fill
3. Spend threshold input focused outline (`border-color: var(--accent)`) — follows global mat-form-field pattern
4. 99th-percentile cap reference line (dashed, accent color, low opacity ~0.4)

Quadrant dot color mapping (from existing semantic palette):
- **Stars** (top-right): `var(--accent)` #FF7700 — scale these
- **Workhorses** (top-left): `var(--warning)` #F39C12 — performing despite low score
- **Question Marks** (bottom-right): `#4285F4` (Google blue, already used for platform-google-ads) — effective creative, underperforming ROAS
- **Laggards** (bottom-left): `var(--text-muted)` #707070 — cut these (desaturated, low priority)

Zero-ROAS dots: plotted at Y=0, filled with `var(--text-muted)` at 60% opacity, no outline — visually distinct from normal plotted dots.

Source: `styles.scss` `:root` and `.dark-theme` / `.light-theme` blocks.

## Layout Contract

### Avg ROAS Stat Tile (entry point)

- Add `cursor: pointer` and hover state: `border-color: var(--accent); box-shadow: var(--shadow-sm)` — mirrors `.asset-tile:hover` pattern
- Add `transition: all var(--transition)` (already defined as `0.2s cubic-bezier(0.4, 0, 0.2, 1)`)
- Add a visual affordance: small `bi-bar-chart-line` icon (12px, `var(--text-muted)`) inline after the label "Avg ROAS" to hint at clickability
- No layout change to the 4-column `agg-stats` grid

### Correlation Drawer

- Width: **560px** — wide enough for a readable scatter chart, leaves content visible on 1280px+ screens
- Position: fixed right, full viewport height, `z-index` above dashboard content
- Background: `var(--bg-card)` with `border-left: 1px solid var(--border)`
- Shadow: `var(--shadow-lg)` on left edge
- Open animation: slide in from right, 200ms `cubic-bezier(0.4, 0, 0.2, 1)` — matches `--transition`
- Backdrop: semi-transparent overlay `rgba(0,0,0,0.4)` behind drawer (click to close)

Drawer internal layout (top to bottom):
1. **Header row** (height 56px): title "Score vs. ROAS" (h4, 16px/500), close button (bi-x, 20px) right-aligned — padding 0 24px
2. **Spend threshold row** (height 64px): label "Min. spend" + outlined number input with "$" prefix + current value display — padding 0 24px
3. **Chart area**: `height: 420px`, full drawer width minus 48px (24px padding each side) — `echart-box` class pattern
4. **Legend row** (height 40px): four quadrant color swatches with labels — padding 0 24px, 16px gap between items

### Scatter Chart

- X-axis: "ACE Score" label, range 0–100, tick marks at 0, 25, 50, 75, 100
- Y-axis: "ROAS" label, range 0 to 99th-percentile cap; ticks auto-generated by ECharts
- Reference lines: vertical at median score + horizontal at median ROAS — both `var(--border)` color (`#404040`), dashed, width 1px
- 99th-percentile cap: horizontal dashed line at cap value, `var(--accent)` at 40% opacity, 1px width
- Quadrant labels: placed in each chart corner at 8px inset from axes, 11px / `var(--text-muted)` / opacity 0.6
- Dot size: fixed radius 6px (no spend-scaling for v1.1 — keeps chart readable at scale)
- Dot hover: radius increases to 9px, shadow `var(--shadow-sm)`
- Grid lines: `var(--border)` at 20% opacity (subtle, ECharts `splitLine.lineStyle.opacity: 0.2`)

### Tooltip (hover on dot)

Container: `var(--bg-card)` background, `border: 1px solid var(--border)`, `border-radius: var(--border-radius)` (8px), `padding: 12px`, `box-shadow: var(--shadow-md)`.

Content layout:
1. Thumbnail: 48×48px, `object-fit: cover`, `border-radius: 4px`, left-aligned
2. Asset name: 14px / 600 / `var(--text-primary)`, max 1 line with ellipsis, beside thumbnail
3. Metric row: Score | ROAS | Spend | Platform — 12px / 400 / `var(--text-secondary)`, inline with `bi-dot` separators

## Interaction Contract

| Trigger | Response |
|---------|----------|
| Click Avg ROAS stat tile | Open correlation drawer (slide in from right) |
| Click drawer backdrop | Close drawer |
| Click bi-x close button | Close drawer |
| Hover scatter dot | Show ECharts tooltip with thumbnail / score / ROAS / spend / platform |
| Click scatter dot | Close drawer, open `AssetDetailDialogComponent` with that asset's data |
| Change spend threshold input | Re-filter scatter data client-side; median reference lines recalculate |
| Dashboard date range or platform filter change | Scatter data refreshes (uses same filtered asset list as the creative grid) |

## States

| State | Visual Treatment |
|-------|-----------------|
| Drawer closed | Avg ROAS tile is default `agg-stat` style; no drawer rendered |
| Drawer opening | 200ms slide-in animation; scatter data fetches simultaneously |
| Chart loading | `skeleton` shimmer div at chart height (420px) — existing `.skeleton` class from styles.scss |
| Chart populated | Scatter dots rendered; reference lines and quadrant labels visible |
| Chart empty (no qualifying assets) | Empty state — centered icon `bi-scatter-chart` (48px, `var(--text-muted)`) + heading + body copy |
| Chart error | Error state — `bi-exclamation-triangle` icon + error copy — matches score-trend-error pattern |
| Dot hovered | Tooltip shown; dot radius 9px; cursor `pointer` |
| Dot clicked | Drawer closes (200ms slide-out); asset detail dialog opens immediately after |
| Spend threshold changed | Chart re-renders with client-side filter; reference lines update; no loading state (synchronous) |
