## Interaction Contract

### Bell icon state machine (primary focal point)

The bell `<button mat-icon-button aria-label="Notifications">` is the primary focal point of this phase.

| State | Visual |
|-------|--------|
| No unread | `bi-bell` icon, no badge |
| 1‚Äì9 unread | `bi-bell` icon + `.custom-badge` showing numeric count |
| 10+ unread | `bi-bell` icon + `.custom-badge` showing `9+` |

Badge position: `top: 4px; right: 4px` (absolute, inside `.notif-btn`).

### Notification menu (MatMenu `#notifMenu`)

- Opens on bell button click (`[matMenuTriggerFor]="notifMenu"`)
- Also opens programmatically via `MatMenuTrigger.openMenu()` when user clicks 'View Notifications' on a toast
- `xPosition="before"` ‚Äî panel opens leftward of bell
- Max visible items: no hard limit; scrollable via MatMenu default overflow
- Empty state: `bi-bell` icon at 24px + opacity 0.4, text "No notifications"
- Network error state: display "Couldn't load notifications. Try again." in place of notification list when `GET /notifications` fails

### Toast (MatSnackBar)

- Triggered by: `SYNC_FAILED` or `TOKEN_EXPIRED` notification with `created_at > lastToastCheckAt`
- One toast per matched notification (no batching)
- Duration: 8000ms
- Action label: "View Notifications" ‚Äî note: D-04 specified the terse label `'View'`; this spec intentionally uses the more descriptive label for clarity. Executor must use "View Notifications".
- Action effect: calls `notifMenuTrigger?.openMenu()` then `loadNotifications()`
- `lastToastCheckAt` updated on every poll tick regardless of whether a toast fired

### Polling

- `setInterval(() => this.loadUnreadCount(), 30000)` ‚Äî existing pattern, extended only
- No SSE. No WebSockets.

### Mark-as-read interactions

- Click any notification item ‚Üí `markRead(n)` ‚Üí `POST /users/notifications/{id}/read`
- Click "Mark all read" ‚Üí `markAllRead()` ‚Üí `POST /users/notifications/read-all`
- Both exist and are implemented. No new interaction needed.
