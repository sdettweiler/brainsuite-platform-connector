## Blocking Issue — UNRESOLVED

**Problem:** When a Meta resync fails due to a revoked token, `sync_status` stays `PENDING` in the DB and no `TOKEN_EXPIRED` notification is created.

**What works:**
- `MetaTokenError` IS raised correctly (code 190 in HTTP 400 response detected)
- Log line `Full resync token error for connection ...` appears correctly
- `_token_err` flag is set in `run_full_resync`

**What fails:**
- The fresh session write after the outer `async with` exits does NOT persist `sync_status = EXPIRED`
- No notification is created
- Frontend polling never stops (stays on "Syncing")

**Root cause investigation so far:**
1. First attempt: `db.add(connection)` after rollback — silent no-op (ORM object expired)
2. Second attempt: `db.execute(UPDATE)` after rollback in same session — `greenlet_spawn` error (nested async session in except block not allowed in SQLAlchemy async)
3. Third attempt: fresh session opened AFTER outer `async with` exits using flag variables `_token_err`, `_terr_conn_id` etc. — still not persisting (unknown why)

**Current code structure in `run_full_resync`** (`backend/app/services/sync/scheduler.py`):
```python
_token_err: Optional[MetaTokenError] = None
_terr_conn_id = None
_terr_job_id = None
_terr_org_id = None
_terr_platform = None

async with get_session_factory()() as db:
    ...
    except MetaTokenError as e:
        _token_err = e
        _terr_conn_id = connection.id
        _terr_job_id = job.id
        _terr_org_id = str(connection.organization_id)
        _terr_platform = connection.platform
        try:
            await db.rollback()
        except Exception:
            pass

if _token_err and _terr_conn_id:
    from sqlalchemy import update as _upd
    async with get_session_factory()() as fresh_db:
        await fresh_db.execute(_upd(PlatformConnection)...values(sync_status="EXPIRED"))
        await fresh_db.execute(_upd(SyncJob)...values(status="FAILED"...))
        await fresh_db.commit()
    asyncio.create_task(create_org_notification(...TOKEN_EXPIRED...))
    return
```

2. Check if `_token_err` flag is actually set (log its value)
3. Check if `Optional` is properly imported (it's used in the type annotation)
4. Verify `SyncJob` is in scope at the post-block location (it's imported inside the `async with` block)
