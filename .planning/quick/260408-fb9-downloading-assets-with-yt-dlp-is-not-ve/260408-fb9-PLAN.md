<objective>
Create a standalone proof-of-concept Python script that tests whether Playwright screen recording can serve as a viable fallback for yt-dlp when downloading YouTube videos.

Purpose: yt-dlp frequently fails on YouTube due to bot detection, age-gating, and sign-in walls. The current codebase uses yt-dlp in both `dv360_sync.py` and `google_ads_sync.py` for YouTube video downloads. This POC evaluates Playwright browser recording as an alternative approach — capturing the video via actual browser playback instead of direct stream download.

Output: A single executable Python script with timing and quality reports.
</objective>

Create a standalone Python script `backend/scripts/poc_playwright_youtube_recorder.py` that:

**1. Argument parsing:**
- Accept a YouTube URL as a positional argument (e.g. `https://www.youtube.com/watch?v=VIDEO_ID`)
- Accept optional `--api-key` flag (falls back to `YOUTUBE_API_KEY` env var)
- Accept optional `--output-dir` flag (defaults to `/tmp/playwright_poc`)
- Accept optional `--headless` flag (default: True for server use, `--no-headless` to watch)

**2. YouTube Data API v3 resolution fetch:**
- Extract video ID from URL using regex
- Call `https://www.googleapis.com/youtube/v3/videos?part=contentDetails,snippet&id={VIDEO_ID}&key={API_KEY}` via httpx
- Parse `contentDetails.definition` (hd/sd) and if available `snippet.thumbnails` to infer aspect ratio
- If API key not provided, fall back to a default 1920x1080 viewport and log a warning
- Time this step separately

**3. Playwright browser recording:**
- Install playwright chromium if needed: run `playwright install chromium` via subprocess at script start if browser not found
- Launch Chromium (headless by default) with `record_video_dir` and `record_video_size` set to the resolved resolution
- Create a new page, set viewport to match
- Navigate to the YouTube URL
- Wait for the video player to load: `page.wait_for_selector('video', timeout=15000)`
- Dismiss cookie consent if present: click `button:has-text("Accept")` with a short timeout, catch if not found
- Click the video to start playback if paused
- Force fullscreen: execute JS `document.querySelector('video').requestFullscreen()` (catch errors)
- Alternatively, press 'f' key on the player for YouTube's native fullscreen
- Get video duration from the DOM: `page.evaluate("document.querySelector('video').duration")`
- Wait for `duration + 3` seconds (buffer for loading)
- Close the page and context to finalize the recording
- Time this step separately

**4. FFmpeg .webm to .mp4 conversion:**
- Find the .webm file in the record_video_dir
- Run `ffmpeg -i input.webm -c:v libx264 -crf 23 -preset fast -c:a aac output.mp4 -y` via subprocess
- Time this step separately

**5. Reports:**
- Print a timing report table:
  ```
  === TIMING REPORT ===
  API fetch:    {x.xx}s
  Recording:    {x.xx}s
  Conversion:   {x.xx}s
  Total:        {x.xx}s
  ```
- Print a quality report:
  ```
  === QUALITY REPORT ===
  Target resolution:  {W}x{H}
  Recorded file:      {filename.webm} ({size} MB)
  Converted file:     {filename.mp4} ({size} MB)
  ```

**Dependencies:** The script should use only: `httpx` (already in requirements.txt), `playwright` (pip install playwright), `subprocess` for ffmpeg, and stdlib. Do NOT add playwright to requirements.txt — this is a POC only.

**Error handling:** Wrap each major step in try/except. If API key missing, warn and use defaults. If playwright fails, print clear error with install instructions. If ffmpeg not found, skip conversion and note it.

**Script must be runnable as:** `python backend/scripts/poc_playwright_youtube_recorder.py "https://www.youtube.com/watch?v=dQw4w9WgXcQ" --api-key YOUR_KEY`
