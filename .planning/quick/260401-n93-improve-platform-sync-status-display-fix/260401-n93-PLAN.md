<objective>
Fix incorrect sync status display for new accounts and add active-sync indicators.

**Problem 1 — New accounts show "Sync Failed":** When an account is first connected, `sync_status` is set to `"ACTIVE"` and `last_synced_at` is `null`. The frontend `getHealthState()` logic falls through to: if `!conn.last_synced_at` and `sync_status !== 'PENDING'` then return `'sync_failed'`. Since the status is `"ACTIVE"` (not `"PENDING"`), new accounts incorrectly show the red "Sync Failed" badge. The `sync-sub` line shows "Syncing..." underneath but only when `!initial_sync_completed` — creating a confusing "Sync Failed + Syncing..." combo.

**Problem 2 — No running sync indicator:** When a user triggers resync, fetch-assets, or initial sync runs in background, there is zero feedback. The UI shows the same stale state. The backend runs `BackgroundTasks` but writes no in-progress state to query.

**Solution:**
1. Use Redis to track in-progress syncs (lightweight, no schema migration).
2. Backend sets `sync:<connection_id>:in_progress` Redis key at start of any sync operation, clears at end.
3. The `/connections` list endpoint bulk-checks Redis for all returned connections and attaches `is_syncing: bool`.
4. Frontend uses `is_syncing` + `initial_sync_completed` + `last_synced_at` to derive correct health state.
5. Frontend polls connections list every 15 seconds when any connection has `is_syncing=true`.

Purpose: Users need accurate, real-time feedback about their account sync status.
Output: Clean status display with correct initial-sync, active-sync, and failure states.
</objective>

Create a helper at the top of scheduler.py:
```python
SYNC_IN_PROGRESS_TTL = 3600  # 1 hour safety TTL — prevents zombie keys if process crashes

async def _set_sync_in_progress(connection_id: str, label: str = "sync"):
    """Mark a connection as actively syncing in Redis."""
    from app.core.redis import get_redis
    redis = get_redis()
    await redis.setex(f"sync:{connection_id}:in_progress", SYNC_IN_PROGRESS_TTL, label)

async def _clear_sync_in_progress(connection_id: str):
    """Clear the syncing flag for a connection."""
    from app.core.redis import get_redis
    redis = get_redis()
    await redis.delete(f"sync:{connection_id}:in_progress")
```

**Fix getHealthState() — the core bug fix:**
```typescript
getHealthState(conn: PlatformConnection): HealthState {
  const now = new Date();
  // Token expired takes priority
  if (conn.token_expiry && new Date(conn.token_expiry) < now) {
    return 'token_expired';
  }
  // Actively syncing (from Redis flag)
  if (conn.is_syncing) {
    // Distinguish initial sync from regular sync
    return conn.initial_sync_completed ? 'syncing' : 'initial_sync';
  }
  // Never synced but not actively syncing = sync error
  if (!conn.last_synced_at) {
    // New account where initial sync hasn't started yet or completed
    if (!conn.initial_sync_completed) {
      return 'initial_sync';  // Just connected, waiting for sync to pick up
    }
    return 'sync_failed';
  }
  // DB-level error status
  if (conn.sync_status === 'ERROR') {
    return 'sync_failed';
  }
  // Stale sync (>48h)
  const hoursSinceSync = (now.getTime() - new Date(conn.last_synced_at).getTime()) / 3_600_000;
  if (hoursSinceSync > 48) {
    return 'sync_failed';
  }
  return 'connected';
}
```
