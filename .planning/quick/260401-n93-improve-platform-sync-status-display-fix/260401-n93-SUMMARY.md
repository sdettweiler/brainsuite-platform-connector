# Quick Task 260401-n93: Improve Platform Sync Status Display Fix

**One-liner:** Redis-based in-progress sync tracking with frontend polling so new accounts show "Initial Sync" instead of "Sync Failed" and active syncs show a live spinner badge.

## Problems Solved

**Problem 1 — New accounts showed "Sync Failed":** `sync_status` is set to `"ACTIVE"` on connect, but `last_synced_at` is `null`. The old `getHealthState()` logic returned `sync_failed` for any account with `!last_synced_at && sync_status !== 'PENDING'`. All new accounts are `ACTIVE`, so they all showed red "Sync Failed" from the moment they connected.

**Problem 2 — No running sync indicator:** Triggering a resync or waiting for initial sync showed zero UI feedback — the stale state badge remained unchanged until a full page reload.

## Solution

### Backend (Task 1)

**scheduler.py:** Added `SYNC_IN_PROGRESS_TTL = 3600`, `_set_sync_in_progress()`, and `_clear_sync_in_progress()` helpers. Instrumented all 4 sync entry points:
- `run_initial_sync` — label `"initial"`, cleared at all exit points (success + every error path including DV360 multi-phase)
- `run_daily_sync` — label `"daily"`, cleared at all exit points
- `run_full_resync` — label `"resync"`, cleared at all exit points
- `run_historical_sync` — label `"historical"`, cleared at all exit points

Safety TTL of 1 hour prevents zombie keys if process crashes mid-sync.

**platforms.py:** `/connections` endpoint now bulk-checks Redis with `mget` for all returned connections and injects `is_syncing: bool` per item. `/connections/{id}/status` endpoint also returns `is_syncing`.

### Frontend (Task 2)

**platforms.component.ts:**
- Added `initial_sync` to `HealthState` type
- Added `is_syncing?: boolean` to `PlatformConnection` interface
- Fixed `getHealthState()`: checks `is_syncing` flag first (returns `syncing` or `initial_sync`); falls back to `initial_sync` for new accounts with no `last_synced_at` and `!initial_sync_completed`; checks `sync_status === 'ERROR'` explicitly; keeps 48h staleness check
- Updated `getHealthLabel()` and `getHealthBadgeClass()` for new states
- Removed redundant `sync-sub` "Syncing..." line from template (now redundant with `initial_sync` state)
- Added `<mat-spinner diameter="12" class="inline-spinner">` inside badge for syncing/initial_sync states
- Added `syncPollInterval` field: starts 15s polling when any `is_syncing=true`, stops when all idle
- `resync()` now reloads connections after 2s delay to pick up the new syncing state immediately
- `ngOnDestroy` clears `syncPollInterval`

**platforms.component.scss:** Added `.badge-syncing` (blue, inline-flex) and `.inline-spinner` (inherits current color via `::ng-deep circle { stroke: currentColor }`) styles.
