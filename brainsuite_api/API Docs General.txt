Each API Clientrepresents an application or integration, and is managed within the Brainsuite Platform. Brainsuite'simplementation is based on the Client Credentials Grant flow of the OAuth 2 specification.Manage API ClientsAll API Clients are listed under Administration → API → API Clients.From this page you can:View an API Client to access its detailsEdit an API Client to update its name or permissions (Admin role required)Delete clients that are no longer in use. This limit helps prevent abuse and denial-of-service attacks, and ensures that the API remainsavailable for all users.The POST endpoints of each app to create or start a job have a primary rate limit. This means thatthere is a limited amount of jobs you can start via API per app. For more information about yourcurrent primary rate limit click here. For all other endpoints a secondary rate limit is applied toensure the quality of service for all users.Rate Limiting detailsTo see your current usage, we provide the following response headers in every POST API response:x-ratelimit-limit: The maximum number of requests you can make in a certain rate limitwindow.x-ratelimit-used: The number of requests you have made in the current rate limit window.x-ratelimit-reset: The time at which the current rate limit window resets, ISO 8601 UTCtimestamp.x-ratelimit-resource: The rate limit resource that the request counted against. Eachapplication has its own rate limits. For more information contact your account manager.Exceeding the rate limitIf you exceed your primary rate limit, you will receive a 429 response, and the x-ratelimit-usedheader will match the x-ratelimit-limit header. You should not retry your request until afterthe time specified by the x-ratelimit-reset header.If your use case requires a higher limit, please contact us and we’ll be happy to discuss your needs.Start a jobA job generates a report of KPIs that assess the effectiveness of an asset. For most tasks, Brainsuite sends aresponse shortly after the system finishes processing the call. This is not always possible with imageand video processing, where times to complete are unpredictable and delayed response can lead totimeout issues. With the Brainsuite Asynchronous API, we introduce asynchronous processing ofstarting a job. Immediately after receiving an asynchronous API request, Brainsuite schedules the joband the system returns a Scheduled status along with a job ID.You can poll later with a different API request to check the status of the original requestcorresponding to the job ID.Asynchronous API WorkflowUsing the asynchronous API follows this poll-based API workflow:Customer client sends a request to the Brainsuite API endpoint to start a job. The asynchronousAPI endpoint responds with the job ID and the current status.Customer client sends the job ID to the Polling API endpoint to find out the updated status of therequest. The Polling API responds with the status of the asynchronous request. If the status iseither Scheduled or Created the customer client continuous to send a request in a specifiedtime interval (e.g. every 30 seconds), until the response status is not Scheduled or Createdanymore.imageError handlingSometimes things go wrong. Here are some tips to help you to handle errors in your API requests.Use the OpenAPI "Try it out" feature of the OpenAPI reference to test your requests. If you arehaving trouble with the API, we recommend using a tool like Postman to get more insights of yourAPI requests.Error Handling StepsReportThe output key in the response of a successful job contains report information and the KPIs. TheExecutive Summary contains a brief overview how the asset performed. The kpis key containsdetailed information for each KPI like visualizations, scores and sometimes additional data.Important note: The visualizations expire after one hour after retrieval of a successful response.More information about the meaning of KPIs can be found in the Academy.Export CockpitBrainsuite enables programmatic export of Cockpit data, which consists of all jobs generated by anApp. The data is available in CSV format and can be exported via the Brainsuite API.The export endpoint is documented in the OpenAPI reference.Asynchronous Export ProcessDue to the potentially large data volume, exports are processed asynchronously.Upon initiating an export, the API returns a file address.The file address can be polled to check the export status. If the file is not yet available, the URLresponds with a 403 Forbidden status. Once the export is complete, the request returns a 200OK status, allowing the file to be downloaded.imageImportant note: The file address expires after one hour.
