# DV360 Bid Manager API: YouTube Campaigns - Comprehensive Research

**Date:** February 27, 2026  
**API Version:** Bid Manager API v2 (Required)  
**Current Status:** Active with ongoing instant reporting migration

---

## 1. OVERVIEW & ARCHITECTURE

### API Endpoints & Structure
- **Primary Service:** Bid Manager API v2 (`https://bidmanager.googleapis.com/v2`)
- **Authentication:** OAuth 2.0 via Google Cloud Project credentials
- **Operations:** Query creation, execution, report retrieval (not resource management)
- **Report Types:** STANDARD, YOUTUBE, REACH_AND_FREQUENCY, URA

### Key Limitation - YouTube & Partners Resources
⚠️ **CRITICAL:** While you can *retrieve* YouTube & Partners resources via the API, you cannot *create* or *update* them programmatically. YouTube campaigns are managed exclusively through:
- Display & Video 360 UI (interactive editing)
- Structured Data Files (bulk upload/manual management)

This differs from standard DV360 line items which can be fully managed via API.

---

## 2. YOUTUBE REPORT TYPES & TEMPLATES

### Available YouTube Report Templates
When `type: "YOUTUBE"` is specified in a query, you can choose templates that determine available metrics:

1. **YouTube Standard Template**
   - Default template for non-conversion metrics
   - Comprehensive performance metrics (impressions, views, VTR, CPV, etc.)
   - Active View measurability metrics
   - Engagement metrics (clicks, CTR)

2. **YouTube Conversion Template**
   - Specialized for conversion measurement
   - Floodlight conversion data integration
   - Post-click and post-view conversions
   - Revenue and ROAS metrics

### Critical Constraint - Conversion vs Non-Conversion Metrics
⚠️ **Cannot query both simultaneously in YouTube reports:**
- You cannot include both conversion metrics and non-conversion metrics in a single YouTube report query
- Conversion metrics will be automatically dropped if you include non-conversion metrics
- Solution: Create separate queries for each metric type

## 6. MAJOR IMPLEMENTATION CONSTRAINTS & GOTCHAS

### A. Reporting Architecture Limitations

1. **No YouTube Resource Management**
   - Cannot create/update YouTube & Partners line items via API
   - Must use UI or Structured Data Files
   - Can only READ YouTube campaign data

2. **Instant Reporting Migration (Ongoing)**
   - Google is migrating from offline to instant reporting
   - Migration happens by partner in batches
   - Check DV360 partner status; YouTube reports now appear in Instant Reporting tab only
   - API access same (`queries.list` endpoint) but source may differ

3. **Real-time vs Batch Data**
   - YouTube reports may have processing delays
   - Not all YouTube metrics available with 1-hour latency
   - Some YouTube-specific dimensions require download reports

### B. Metric Specificity Issues

1. **YouTube vs Display Metrics Not Comparable**
   - Video view rates across platforms differ fundamentally
   - Meta 15-second ThruPlay ≠ TikTok 6-second Focused View ≠ YouTube view
   - Cannot use Meta API's `video_15_sec_watched_actions` directly in YouTube reports
   - Need separate normalization logic in Platform Connector MVP

2. **Viewability Measurement Differences**
   - YouTube uses TrueView-specific viewability (50%+ 2+ seconds for video)
   - Active View metrics available but computed differently than display
   - `METRIC_ACTIVE_VIEW_AUDIBLE_FULLY_ON_SCREEN_HALF_OF_DURATION_IMPRESSIONS` is YouTube-specific

3. **Conversion Attribution Model Limitations**
   - Cannot specify attribution model in API (unlike UI)
   - Floodlight required for conversion tracking
   - Default is last-touch for post-click; post-view attribution is 1-day lookback
   - Cannot customize conversion window in API queries

### C. Rate Limiting & Quota

- **Default quota:** 100 requests per day per partner
- **Report execution:** Synchronous via `queries.run()`
- **Large datasets:** Use exponential backoff for polling report status
- **YouTube reports take longer** than standard reports to process

### D. Deprecated Elements (Important for 2025+)

- **October 1, 2024 changes (already implemented):**
  - `FILTER_ANONYMOUS_INVENTORY_MODELING` sunset
  - Total Media Cost metrics incompatible with YouTube
  - DST/ROC fees incompatible with YouTube

- **March 1, 2025 (upcoming):**
  - YouTube TrueView Inventory Availability report deprecated
  - Related dimensions being removed
  - Use planning/forecasting features in UI instead

## 8. IMPLEMENTATION RECOMMENDATIONS FOR BRAINSUITE PLATFORM CONNECTOR MVP

### A. Data Integration Strategy

**For YouTube Campaign Data:**
1. Use Bid Manager API v2 for reporting (read-only)
2. Use Display & Video 360 API v4 (or Structured Data Files) for resource metadata
3. **Separate query execution for:**
   - Video performance metrics (non-conversion)
   - Conversion metrics (if Floodlight enabled)

**Metric Normalization Challenges:**
- YouTube views ≠ Meta video watches ≠ TikTok focused views
- Create unified "Video Performance Score" with weights:
  - Weight video views, first quartile, mid-point, and complete
  - Normalize across platforms using platform-specific baseline
  - Document these transformations in integration spec

### B. Field Mapping for Platform Connector

|---|---|---|
| METRIC_VIDEO_VIEWS | video_views | Platform-specific |

### D. Error Handling Strategy

**Expected error scenarios:**

1. **400 Bad Request** - Check for:
   - Incompatible metric/filter combinations
   - Deprecated filter values
   - Mixing conversion + non-conversion metrics
   - Total Media Cost or Fee32 metrics with YOUTUBE type

2. **403 Forbidden** - Verify:
   - OAuth token has correct advertiser scope
   - User has YouTube line item access
   - Not using partner-level credentials for YouTube data

3. **Report Processing Timeout** - Implement:
   - Exponential backoff polling (start 30s, max 5min intervals)
   - Query timeout after 24 hours
   - Fall back to instant reporting if available

## 12. SUMMARY OF CRITICAL FINDINGS

### ✅ What Works Well for YouTube Integration
- Comprehensive video performance metrics (views, view rate, CPV)
- Extensive demographic targeting dimensions
- Active View viewability data (MRC-accredited)
- Conversion tracking via Floodlight
- Instant reporting (newly available)

### ⚠️ Key Challenges for Multi-Platform Harmonization
1. **Metric Definitions Differ Significantly**
   - YouTube view ≠ Meta video watch ≠ TikTok view
   - Requires custom normalization in Platform Connector

2. **API Limitations**
   - Cannot create/update YouTube resources programmatically
   - Conversion + non-conversion metrics cannot coexist in one query
   - Some metrics incompatible with YOUTUBE report type

3. **Data Latency**
   - YouTube reports process slower than standard display reports
   - Instant reporting still migrating (batch availability varies by partner)

### 🎯 Recommended MVP Implementation Scope
- **Phase 1:** Video performance metrics (views, CTR, CPV) for non-conversion focused line items
- **Phase 2:** Audience demographic breakdowns (age, gender, geography)
- **Phase 3:** Conversion tracking (requires Floodlight configuration)
- **Future:** YouTube Analytics API integration for video-specific insights beyond campaign metrics
